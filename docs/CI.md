# GitHub Actions

Workflows run on `ubuntu-latest` using Node 24–compatible actions:

| Action | Version |
|--------|---------|
| `actions/checkout` | v5 |
| `actions/setup-python` | v6 (pip cache enabled) |
| `azure/login` | v2 |
| `hashicorp/setup-terraform` | v3 |
| `actions/cache` | v4 |

## Secrets

Configure under **Settings → Secrets and variables → Actions**.

| Secret | Required by | Purpose |
|--------|-------------|---------|
| `AZURE_CREDENTIALS` | `deploy.yml` | JSON service principal for `azure/login` (`clientId`, `clientSecret`, `subscriptionId`, `tenantId`). Create manually — see below. |
| `BASILISK_TOKEN_SECRET` | `ci.yml`, `e2e.yml` | HMAC secret for HKP v2 bearer tokens in tests. **Terraform generates the production value** — copy from `export-github-secrets` after first deploy. Workflows fall back to `ci-test-secret` if unset. |

Optional repository secrets (workflow reads Terraform state when these are omitted):

| Secret | Purpose |
|--------|---------|
| `AZURE_SUBSCRIPTION_ID` | Pin subscription for deploy |
| `BASILISK_NAME_PREFIX` | Default name prefix when not passed as workflow input |

## Bootstrap: first deploy + GitHub secrets

### 1. Deploy infrastructure (local or Cloud Shell)

```bash
az login
./scripts/deploy-terraform-cloudshell.sh
# or with auto-approve:
AUTO_APPROVE=true ./scripts/deploy-terraform-cloudshell.sh
```

### 2. Export Terraform-generated secrets

```bash
bash scripts/export-github-secrets.sh
```

PowerShell:

```powershell
.\scripts\export-github-secrets.ps1
```

This prints:

- **`BASILISK_TOKEN_SECRET`** — generated by Terraform (`random_password`), already set on the Function App
- **`AZURE_CREDENTIALS` command** — run locally to create the deploy service principal
- Optional metadata (resource group, function name, Front Door URL)

Paste into GitHub:

```bash
gh secret set BASILISK_TOKEN_SECRET --body "<value from script>"
gh secret set AZURE_CREDENTIALS --body '<json from az ad sp create-for-rbac --sdk-auth>'
```

Or use **Settings → Secrets and variables → Actions → New repository secret**.

### 3. Create `AZURE_CREDENTIALS`

The export script prints a scoped command, e.g.:

```bash
az ad sp create-for-rbac \
  --name basilisk-github-deploy \
  --role contributor \
  --scopes /subscriptions/<sub-id>/resourceGroups/basilisk-dev-rg \
  --sdk-auth
```

Paste the full JSON output into the `AZURE_CREDENTIALS` secret.

### 4. Deploy via GitHub Actions

**Actions → deploy → Run workflow**

| Input | Default | Notes |
|-------|---------|-------|
| `name_prefix` | `basilisk-dev` | Must match Terraform deploy |
| `location` | (empty) | Auto-detect from existing RG |
| `mail_provider` | `office365` | Logic App connector |
| `skip_terraform` | `false` | Set `true` for code-only redeploys |

The workflow:

1. Applies Terraform (infra + app settings including `BASILISK_TOKEN_SECRET`)
2. Zips and publishes the Function App (`az functionapp deploy`)
3. Smoke-tests Front Door `/health`

Terraform state is cached per `name_prefix` in GitHub Actions. For team/production use, configure remote state — see `terraform/cloudshell/backend.tf.example`.

## Local vs CI

- **Never commit** `.env`, `.env.test`, or `local.settings.json` (see `.gitignore`).
- **Committed templates:** `.env.example`, `.env.test.example`
- **CI:** `scripts/prepare-ci-env.sh` copies examples and injects `BASILISK_TOKEN_SECRET` from GitHub Secrets when set.

## Generate values manually

```bash
# Token secret (if not using Terraform output)
python -c "import secrets; print(secrets.token_urlsafe(48))"
```
